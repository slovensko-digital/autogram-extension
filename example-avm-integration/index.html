Autogram v mobile integration demo

Example AVM Integration

This example follows the same lightweight Vite pattern as example-extension-usage, but drives a real AVM integration flow: register or reuse an integration, upload a document, generate a mobile deep-link or QR payload, and wait for the signed file.

1. Integration

The integration key pair and GUID are persisted in IndexedDB so the same demo deployment can be paired with the mobile app once and reused.

Device pairing QR

Scan with Autogram v mobile to pair this integration with the app. Regenerated each time you load or register.

2. Document

Upload a file or use inline text content. Text payloads are sent as plain text, binary files are sent as base64 with the matching MIME type.

3. Mobile app handoff

Open the generated URL on a mobile device with Autogram v mobile, or scan the QR code. If the integration has already been paired, the same wait operation also triggers a sign request push.

With integration JWT

AVM will try to register this integration with the mobile app for push notifications.

Without integration JWT

Plain signing URL — no pairing, no push notifications.

4. Wait for signature

The SDK sends a sign-request to paired devices and polls the document until it changes to a signed state.
