Podpisovac akceptuje v sucasnosti 2 typy requestov:

XML data -> XAdES Asice + zabalenie contentu ako XMLDataContainer (pre ucely komunikacie s verejou spravou). Na vstupe ocakava:

  • XML subor (content na podpis)
  • XSD schema (v parametroch)
  • XSLT transformacia (v parametroch)
  • ostatne parametre (najma identifikator a verzia)

PDF data -> PAdES - zatial s podpisom bez vizualneho efektu v dokunente. Na vstupe ocakava:

  • PDF subor (base64) (content na podpis)
  • ostatne parametre
V blizkej buducnosti ocakavana implementacia templatov pre parametre (slim requesty).

  6. Poslat request s PDF (base64) a PADES parametrami + vizualizacia PDF

Tipy:

- requesty prijima na url http://127.0.0.1:37200/sign

- odporucam pri testovani sledovat browser dev tools (console a network) pre presnejsie identifikovanie chyby

Zname nedostatky:
    Zname chyby ktore sa budu odstranovat njblizsie:
  • nemoznost podpisat nekompletny request
  • malo moznosti customizacie requestov
  • pravdepodobna nefunkcnost query parametra ('?template=sk' - moznost slobodne omitnut vacsinu low-level settingov z request body)
  • datakontainer nema prefix xdc
